# Архитектура

## Принятое решение

Бэкенд — отдельный сервис (Go или Node.js).  
Next.js — только фронтенд.

---

## Клиент (Next.js)

- UI: React (App Router)
- Формы: React-Hook-Form
- Работа с данными и кэширование: TanStack Query
- Визуализация генеалогического древа: клиентский компонент (выбранная библиотека)

### Ответственность клиента:
- Отображение данных
- Отправка запросов к API
- Простая валидация форм (UI-уровень)
- Управление состоянием интерфейса

На клиенте **нет** бизнес-логики родства и проверки связей.

---

## Бэкенд (Go / Node.js)

Отдельный API-сервис.

### Ответственность бэкенда:

- Регистрация / авторизация
- Управление деревом (в MVP — одно дерево на пользователя)
- CRUD персон
- Проверка корректности связей (отсутствие циклов)
- Правила удаления персон
- Расчёт степени родства относительно корневой персоны ("Я")
- Работа с БД и миграциями

Бэкенд — единственный источник истины.

---

## Поток данных

UI  
→ TanStack Query  
→ Backend API (https://api.домен/...)  
→ Ответ  
→ Отображение в UI

---

## Правила архитектуры

1. Бизнес-логика находится только на бэкенде.
2. Клиент не вычисляет родство.
3. Клиент не проверяет граф на циклы.
4. Клиент не принимает архитектурные решения.
5. Все данные проходят через API.
