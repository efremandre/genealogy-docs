# API Endpoints — Объяснение

## AUTH

### POST /api/auth/register

**Назначение:**
Создание нового пользователя.

**Что происходит:**
- Проверяется, существует ли email
- Пароль хэшируется
- Создаётся запись в таблице User
- Создаётся пустое Tree для пользователя
- Возвращается базовая информация о пользователе

**Тип запроса:**
POST — создаётся новый ресурс.

### POST /api/auth/login

**Назначение:**
Аутентификация пользователя.

**Что происходит:**
- По email ищется пользователь
- Пароль сравнивается с хэшем
- Создаётся сессия или выдаётся токен
- Возвращается объект пользователя

Это вход в систему.

### POST /api/auth/logout

**Назначение:**
Завершение сессии пользователя.

**Что происходит:**
- Удаляется сессия / инвалидируется токен

**Возвращается:**

```json
{ "ok": true }
```

## TREE

### GET /api/tree

**Назначение:**
Получить дерево текущего пользователя.

**Что происходит:**
- По сессии определяется пользователь
- Возвращается его дерево
- В MVP дерево одно.

### PATCH /api/tree

**Назначение:**
Обновление параметров дерева.

**Можно менять:**
- title
- rootPersonId (кто считается "Я")

**Тип запроса:**
PATCH — частичное обновление данных.


## PERSONS

### GET /api/persons

**Назначение:**
Получить всех персон текущего дерева.

**Используется для:**
- списка персон
- построения визуализации
- формирования структуры графа

### POST /api/persons

**Назначение:**
Создание новой персоны.

**Принимает:**

```json
{
  "firstName": "string",
  "lastName": "string",
  "gender": "string | null",
  "birthYear": 1900,
  "fatherId": "uuid | null",
  "motherId": "uuid | null"
}
```

**Важно:**
- fatherId и motherId должны ссылаться на персон того же дерева

### PATCH /api/persons/:id

**Назначение:**
Редактирование существующей персоны.

**Можно менять:**
- имя
- фамилию
- год рождения
- родителей

**Важно:**
- Нельзя допускать циклических связей
- Нельзя назначать более двух родителей

### DELETE /api/persons/:id

**Назначение:**
Удаление персоны.

**Бизнес-логика (выбрать один вариант):**
- Запретить удаление, если есть дети
- Либо автоматически отвязать её как родителя у детей


## RELATION

### GET /api/relation/:personId

**Назначение:**
Получить степень родства относительно корневой персоны.

**Пример ответа:**

```json
{
  "relationLabel": "прадед"
}
```

**Важно:**
- Родство не хранится в базе
- Родство вычисляется алгоритмом на основе связей
- Это ключевая логика продукта
